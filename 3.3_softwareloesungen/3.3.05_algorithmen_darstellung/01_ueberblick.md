# 3.3.05 – Algorithmen in einer Programmiersprache darstellen

## Einordnung

Dieses Thema gehoert zum Bereich **3.3 Programmieren von Softwareloesungen** und behandelt die Darstellung von Algorithmen in **Pseudocode** und Code-naher Notation. In der IHK-Pruefung muss Code **nicht 1:1 kompilierbar** sein – es geht darum, dass der Algorithmus **verstaendlich** und die **Kontrollstrukturen ersichtlich** sind.

## Themenkreise

| Nr. | Themenkreis | Schwerpunkt |
|-----|-------------|-------------|
| 1 | Allgemein verstaendlicher Pseudocode | Konventionen, klare Formulierung |
| 2 | Code muss nicht 1:1 kompilierbar sein | Fokus auf Logik, nicht auf Syntax |
| 3 | Kontrollstrukturen ersichtlich | Schleifen, Verzweigungen, Sequenzen |

## Pruefungsrelevanz

- **Haeufig gefragt**: Algorithmen lesen und verstehen, Pseudocode schreiben
- **Typische Aufgaben**: Code-Traces (="Was gibt der Code aus?"), Algorithmus in Pseudocode formulieren
- **Wichtig**: Einrueckung und Struktur muessen klar sein

## Lernziele

Nach Bearbeitung dieses Themas solltest du:
1. Pseudocode lesen und selbst schreiben koennen
2. Kontrollstrukturen (Sequenz, Selektion, Iteration) korrekt darstellen koennen
3. Code-Traces (schrittweises Durchrechnen) ausfuehren koennen
4. Einfache Algorithmen (Sortierung, Suche) in Pseudocode formulieren koennen

# 3.3.03 – Festlegen von Schnittstellen

## Einordnung

Dieses Thema gehoert zum Bereich **3.3 Programmieren von Softwareloesungen** und behandelt die Definition und Nutzung von Schnittstellen (Interfaces/APIs) in der Softwareentwicklung. Im Fokus stehen **REST-APIs**, **Datenaustauschformate** und **SQL** als Schnittstelle zur Datenbank.

## Themenkreise

| Nr. | Themenkreis | Schwerpunkt |
|-----|-------------|-------------|
| 1 | API (REST) | RESTful Services, HTTP-Methoden, Ressourcen, Statuscode |
| 2 | Datenaustauschformate (XML, JSON) | Syntax, Vor-/Nachteile, Einsatzgebiete |
| 3 | SQL | SQL als Datenschnittstelle, CRUD-Operationen |

## Pruefungsrelevanz

- **Haeufig gefragt**: REST-Prinzipien, JSON-Syntax, HTTP-Statuscodes
- **Typische Aufgaben**: API-Endpunkte definieren, JSON/XML lesen und schreiben, SQL-Abfragen
- **Verknuepfung**: Netzwerktechnik, Datenbanken, Webentwicklung

## Querverweise

- **2.2.03** – SQL-Abfragen

## Lernziele

Nach Bearbeitung dieses Themas solltest du:
1. Die REST-Prinzipien erklaeren und API-Endpunkte entwerfen koennen
2. JSON und XML lesen, schreiben und vergleichen koennen
3. HTTP-Methoden und Statuscodes korrekt zuordnen koennen
4. SQL als Datenschnittstelle einsetzen koennen

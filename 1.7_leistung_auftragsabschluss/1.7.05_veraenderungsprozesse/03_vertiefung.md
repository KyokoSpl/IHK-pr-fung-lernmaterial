# Vertiefung: 1.7.05 – Veraenderungsprozesse / Change Management

## Vergleich: Rollen im Veraenderungsprozess

| Merkmal | Promoter | Skeptiker | Bremser | Widerstaendler |
|---|---|---|---|---|
| **Einstellung** | Positiv | Kritisch-konstruktiv | Passiv-negativ | Aktiv-negativ |
| **Verhalten** | Treibt voran | Hinterfragt | Verzoegert | Blockiert |
| **Wert** | Motor des Wandels | Qualitaetssicherung | Fruehwarnung | Zeigt Konflikte |
| **Strategie** | Einsetzen als Multiplikator | Ernst nehmen, einbinden | Motivieren, Ursachen klaeren | Gespraeche, ggf. Konsequenzen |

## Zusammenhang: Lewin-Modell in der IT

```
Unfreeze                    Change                     Refreeze
├─ Probleme aufzeigen       ├─ Neues System einfuehren ├─ SOPs/Handbuecher
├─ Schulungsbedarf          ├─ Schulungen               ├─ Feedback-Runden
│  ermitteln                ├─ Begleitung                ├─ Optimierung
├─ Kommunikation            ├─ Quick Wins zeigen        ├─ Neuer Standard
└─ Widerstaende adressieren └─ Support bereitstellen    └─ Erfolgsmessung
```

## Typische Pruefungsaspekte

1. Rollen (Promoter, Skeptiker, Bremser, Widerstaendler) zuordnen
2. Ursachen fuer Widerstaende benennen
3. Massnahmen zur Widerstandsreduktion vorschlagen
4. 3-Phasen-Modell nach Lewin beschreiben
5. Blended Learning erklaeren

## Haeufige Fehler

| Fehler | Richtig |
|---|---|
| Skeptiker = Widerstaendler | Skeptiker sind konstruktiv-kritisch, Widerstaendler blockieren aktiv |
| Widerstand ist immer negativ | Widerstand kann auf echte Probleme hinweisen |
| Change Management = Software-Rollout | CM umfasst auch menschliche/organisatorische Aspekte |
| Kommunikation ist nebensaechlich | Kommunikation ist DER Erfolgsfaktor |

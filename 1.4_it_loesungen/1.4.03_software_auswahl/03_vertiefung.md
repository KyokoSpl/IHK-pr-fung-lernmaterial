# Vertiefung: 1.4.03 – Bedarfsgerechte Auswahl von Software

## Open Source vs. Proprietaer – Detailvergleich

| Aspekt | Open Source | Proprietaer |
|---|---|---|
| Quellcode | Offen, einsehbar | Geschlossen |
| Kosten | Kostenlos (Lizenz) | Lizenzgebuehren |
| Support | Community, optional kommerziell | Professionell (Hersteller) |
| Anpassbarkeit | Hoch (Quellcode aenderbar) | Gering (nur Konfiguration) |
| Sicherheit | Transparent (viele Augen) | Intransparent, abhaengig vom Hersteller |
| Vendor Lock-in | Kein Lock-in | Hoch |
| Schulungen | Eigeninitiatve/Community | Zertifizierte Schulungen |

## Entscheidungshilfe: Standard vs. Individual

**Standardsoftware waehlen, wenn:**
- Anforderungen weitgehend durch Standardfunktionen abgedeckt
- Budget begrenzt
- Schnelle Einfuehrung gewuenscht
- Hersteller-Support gewuenscht

**Individualsoftware waehlen, wenn:**
- Einzigartige Geschaeftsprozesse
- Keine Standardsoftware deckt die Anforderungen
- Langfristige Investition geplant
- Wettbewerbsvorteil durch einzigartige Software

## KI-Einsatz – Chancen und Risiken

| Chancen | Risiken |
|---|---|
| Effizienzsteigerung | Datenschutzprobleme (DSGVO) |
| Automatisierung repetitiver Aufgaben | Fehlentscheidungen durch Bias |
| Verbesserte Datenanalyse | Abhaengigkeit von KI-Anbietern |
| Personalisierte Nutzerinteraktion | Ethische Bedenken |

## Zusammenhang zu anderen Themen
- **Lizenzmodelle** (1.4.04): GPL, MIT, EULA – rechtliche Grundlage fuer Open Source und proprietaer
- **Wirtschaftlichkeit** (1.3.04): Vergleich Lizenzkosten Abo vs. Einmalkauf vs. Open Source
- **Programmiersprachen** (1.4.06): IDEs als Werkzeug fuer Entwicklung

## Typische Pruefungsaspekte
- Szenario: Welche Software fuer welchen Einsatzzweck?
- Vor-/Nachteile Open Source vs. proprietaer
- Standard- vs. Individualsoftware begruendet empfehlen
- IDE-Bestandteile benennen
- KI-Einsatz im Unternehmen bewerten

## Haeufige Fehler
- Open Source = kostenlos und ohne Regeln → Lizenzen beachten!
- Standardsoftware kann NICHT immer angepasst werden → Konfiguration ≠ Modifikation
- IDE = nur Editor → IDE buendelt mehrere Werkzeuge

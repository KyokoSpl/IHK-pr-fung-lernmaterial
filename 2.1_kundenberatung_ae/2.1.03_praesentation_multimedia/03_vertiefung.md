# Vertiefung: 2.1.03 – Daten multimedial aufbereiten und praesentieren

## Zusammenhang der Themenkreise

```
Corporate Identity (CI)
        |
        v
+------------------+     +-------------------+
| Praesentations-  |---->| Grafische         |
| programm         |     | Darstellung       |
| (PowerPoint etc.)|     | (Diagramme, Bilder)|
+------------------+     +-------------------+
        |                        ^
        v                        |
+------------------+     +-------------------+
| Praesentations-  |     | Tabellenkalkulation|
| technik          |     | (Excel, Calc)      |
| (Vortrag)        |     | → Daten → Diagramm |
+------------------+     +-------------------+
        |                        |
        v                        v
+----------------------------------------+

| (Videos, Screencasts, Infografiken)    |
+----------------------------------------+
        |
        v
  Visualisierung
  (Zielgruppe + Daten → passende Form)
```

## Typische Pruefungsaspekte

- Richtiges Diagramm fuer eine gegebene Datenreihe waehlen
- Excel-Formeln anwenden (SUMME, WENN, SVERWEIS)
- CI-Vorgaben bei Praesentationen beachten
- Vor-/Nachteile verschiedener Dateiformate (JPEG vs. PNG vs. SVG)
- Praesentationsregeln anwenden

## Vergleich: Diagrammtypen

| Kriterium | Saeulendiagramm | Kreisdiagramm | Liniendiagramm |
|---|---|---|---|
| Datentyp | Kategorien | Anteile | Zeitreihen |
| Max. Kategorien | 10-15 | 5-7 | Unbegrenzt |
| Vergleich | Gut | Schlecht | Mittel |
| Trend erkennen | Mittel | Nein | Sehr gut |
| Negative Werte | Ja | Nein | Ja |

## Haeufige Fehler in Pruefungen

| Fehler | Richtig |
|---|---|
| Kreisdiagramm fuer Zeitreihe | Liniendiagramm verwenden |
| Y-Achse nicht bei 0 begonnen | Verzerrung vermeiden, bei 0 starten |
| 3D-Diagramme ohne Grund | 2D ist klarer und praeziser |
| Zu viele Animationen | Maximal 1-2 pro Folie, dezent |
| Fehlende Achsenbeschriftung | Immer Einheiten und Beschreibung angeben |
| SVERWEIS ohne FALSCH | Exakte Suche: letzter Parameter FALSCH |

# 4.5.03 – Ethische Aspekte und Compliance-Regelungen: Grundlagen

## TK 1: Allgemeine und betriebliche Compliance-Regelungen

**Definition:** Compliance bezeichnet die Einhaltung von gesetzlichen Vorschriften, regulatorischen Standards und internen Unternehmensrichtlinien. Ein Compliance-Management-System (CMS) ist die strukturierte Gesamtheit aller Massnahmen, um Regeltreue sicherzustellen.

### Elemente eines Compliance-Management-Systems (CMS)

| Element | Beschreibung | Beispiel |
|---------|-------------|----------|
| Compliance-Kultur | Werte und Haltung der Unternehmensleitung | Tone from the Top, Vorbildfunktion |
| Compliance-Ziele | Definierte Ziele der Regeltreue | Null-Toleranz bei Korruption |
| Compliance-Risiken | Identifikation und Bewertung von Risiken | Datenschutzverstoesse, Bestechung |
| Compliance-Programm | Massnahmen zur Risikominimierung | Schulungen, Richtlinien, Kontrollen |
| Compliance-Organisation | Verantwortlichkeiten und Strukturen | Compliance-Officer, Meldekanal |
| Compliance-Kommunikation | Information und Schulung der Mitarbeitenden | E-Learning, Aushang, Intranet |
| Compliance-Monitoring | Ueberwachung und Verbesserung | Audits, Kennzahlen, Berichte |

### Verhaltenskodex (Code of Conduct)

| Bereich | Typische Regelung |
|---------|-------------------|
| Anti-Korruption | Geschenke nur bis Wertgrenze (z.B. 35 EUR), Dokumentationspflicht |
| Datenschutz | DSGVO-konformer Umgang mit personenbezogenen Daten |
| IT-Sicherheit | Sichere Passwoerter, keine private Software, Meldepflicht |
| Gleichbehandlung | AGG-Konformitaet, keine Diskriminierung |
| Interessenkonflikte | Offenlegung von Nebentaetigkeiten, Beteiligungen |
| Umwelt | Ressourcenschonung, Einhaltung von Umweltvorschriften |
| Soziale Verantwortung | Faire Arbeitsbedingungen, Lieferketten-Sorgfalt |

### Whistleblowing und Hinweisgeberschutzgesetz (HinSchG)

| Aspekt | Regelung |
|--------|----------|
| Geltungsbereich | Unternehmen ab 50 Mitarbeitenden |
| Meldekanaele | Interne Meldestelle (Pflicht), externe (BaFin, Kartellamt etc.) |
| Schutz | Schutz vor Repressalien (Kuendigung, Versetzung, Mobbing) |
| Vertraulichkeit | Identitaet des Hinweisgebers wird geschuetzt |
| Frist | Rueckmeldung innerhalb von 3 Monaten |
| Beweislastumkehr | Arbeitgeber muss beweisen, dass Massnahme nicht wegen Meldung erfolgte |
| Sanktionen | Bis zu 50.000 EUR Bussgeld bei Verstoss gegen HinSchG |

### Allgemeines Gleichbehandlungsgesetz (AGG)

| Merkmal | Geschuetzt vor Diskriminierung |
|---------|-------------------------------|
| Rasse / ethnische Herkunft | Rassistische Diskriminierung |
| Geschlecht | Geschlechtsbezogene Benachteiligung |
| Religion / Weltanschauung | Religioese Diskriminierung |
| Behinderung | Benachteiligung aufgrund von Behinderung |
| Alter | Altersdiskriminierung |
| Sexuelle Identitaet | Diskriminierung aufgrund sexueller Orientierung |

## TK 2: Gender-Neutralitaet (drittes Geschlecht)

**Definition:** Gender-Neutralitaet bezeichnet die Gleichbehandlung aller Geschlechtsidentitaeten, einschliesslich des seit 2018 in Deutschland anerkannten dritten Geschlechts ("divers", § 22 Abs. 3 PStG).

### Geschlechtsidentitaeten im rechtlichen Kontext

| Eintrag im Personenstandsregister | Bedeutung |
|-----------------------------------|-----------|
| maennlich | Maennliche Geschlechtszuordnung |
| weiblich | Weibliche Geschlechtszuordnung |
| divers | Drittes Geschlecht (seit 2018) |
| keine Angabe | Offen lassen (seit 2018) |

### Gender-gerechte Sprache in der IT

| Methode | Beispiel | Anwendung |
|---------|----------|-----------|
| Doppelnennung | Entwicklerinnen und Entwickler | Formelle Texte, Vertraege |
| Gender-Stern | Entwickler*innen | Intern, wenn gewuenscht |
| Gender-Doppelpunkt | Entwickler:innen | Barrierefreier als Stern (Screenreader) |
| Neutralisierung | Entwickelnde, Fachkraefte, Team | Techische Dokumentation |
| Partizip | Studierende, Mitarbeitende | Offizielle Texte |

### Relevanz fuer IT-Systeme

| Bereich | Anforderung |
|---------|-------------|
| Formularfelder | Anrede: "Herr", "Frau", "Divers" oder freies Textfeld |
| Datenbank-Design | Geschlecht als ENUM mit mindestens 3+ Optionen oder Freitext |
| E-Mail-Vorlagen | Gender-neutrale Anrede ("Guten Tag [Vorname Nachname]") |
| Benutzerprofile | Pronomen-Feld (er/sie/they etc.) optional anbieten |
| Algorithmen | Keine geschlechtsspezifischen Annahmen in Empfehlungen |

## TK 3: Nachhaltigkeit

**Definition:** Nachhaltigkeit in der IT umfasst die Beruecksichtigung oekologischer, oekonomischer und sozialer Auswirkungen von IT-Entscheidungen auf Umwelt, Lebensraeume und zukuenftige Generationen.

### Drei Saeulen der Nachhaltigkeit in der IT

| Saeuele | Beschreibung | Beispiel |
|---------|-------------|----------|
| Oekologisch | Umweltauswirkungen minimieren | Energieeffizienz, Recycling, CO2-Reduktion |
| Oekonomisch | Langfristige Wirtschaftlichkeit | Total Cost of Ownership, Lebensdauer |
| Sozial | Gesellschaftliche Verantwortung | Faire Lieferketten, Arbeitsbedingungen |

### Green IT – Konkrete Massnahmen

| Bereich | Massnahme | Auswirkung |
|---------|-----------|------------|
| Hardware | Laengere Nutzungsdauer, Refurbished-Geraete | Weniger Elektroschrott |
| Rechenzentren | Energieeffiziente Kuehlung, Oekostrom | CO2-Reduktion |
| Software | Effizienter Code, weniger Ressourcenverbrauch | Geringerer Energiebedarf |
| Cloud | Shared Resources statt eigener Server | Bessere Auslastung |
| Drucken | Papierloses Buero, beidseitiger Druck | Ressourcenschonung |
| Beschaffung | Umweltlabel beachten (Blauer Engel, Energy Star) | Nachhaltige Lieferkette |
| Entsorgung | Fachgerechtes Recycling (ElektroG/WEEE) | Rohstoffrueckgewinnung |

### Umweltauswirkungen der IT-Branche

| Aspekt | Problem | Loesungsansatz |
|--------|---------|---------------|
| Energieverbrauch | Rechenzentren verbrauchen ca. 1-2% des weltweiten Stroms | Erneuerbare Energien, effiziente Hardware |
| Seltene Erden | Abbau schaedigt Lebensraeume | Recycling, laengere Nutzung, modulare Bauweise |
| Elektroschrott | 50+ Mio. Tonnen pro Jahr weltweit | Reparierbarkeit, Leasing-Modelle |
| CO2-Emissionen | Streaming, Cloud-Computing, Blockchain | Optimierung, bewusster Konsum |
| Wasserverbrauch | Kuehlung von Rechenzentren | Alternative Kuehlkonzepte |

## TK 4: Wuerde und Integritaet aller Menschen

**Definition:** Bei der Entwicklung und dem Einsatz von IT-Loesungen muessen die Wuerde und Integritaet aller direkt und indirekt betroffenen Menschen gewahrt werden – einschliesslich derjenigen, deren Daten verarbeitet oder deren Entscheidungen durch Algorithmen beeinflusst werden.

### KI-Ethik: Grundprinzipien

| Prinzip | Beschreibung | Beispiel |
|---------|-------------|----------|
| Fairness | Keine Diskriminierung durch Algorithmen | KI-Bewerbungstool benachteiligt keine Geschlechter |
| Transparenz | Entscheidungen nachvollziehbar machen | Erklaerbare KI (Explainable AI, XAI) |
| Verantwortlichkeit | Klare Zustaendigkeit fuer KI-Entscheidungen | Menschliche Letztentscheidung (Human in the Loop) |
| Datenschutz | Schutz personenbezogener Daten | Privacy by Design, Datenminimierung |
| Sicherheit | Robustheit gegen Manipulation | Adversarial Attacks erkennen und abwehren |
| Nicht-Schaedigung | Kein Schaden fuer Menschen | Autonome Systeme mit Sicherheitsmechanismen |

### Bias in KI-Systemen

| Bias-Art | Beschreibung | Beispiel |
|----------|-------------|----------|
| Historischer Bias | Trainingsdaten spiegeln vergangene Diskriminierung | Weniger Frauen in IT-Positionen → KI bevorzugt Maenner |
| Repraesentations-Bias | Bestimmte Gruppen unterrepraesentiert | Gesichtserkennung funktioniert schlechter bei dunkler Haut |
| Mess-Bias | Fehlerhafter Proxy fuer das Ziel | Postleitzahl als Proxy korreliert mit Ethnie |
| Aggregations-Bias | Eine Loesung fuer heterogene Gruppen | Medizin-KI trainiert hauptsaechlich an maennlichen Patienten |

### Barrierefreiheit – Standards

| Standard | Beschreibung | Geltungsbereich |
|----------|-------------|-----------------|
| WCAG 2.1 | Web Content Accessibility Guidelines (W3C) | International, Web |
| BITV 2.0 | Barrierefreie-Informationstechnik-Verordnung | Deutschland, oeffentliche Stellen |
| EN 301 549 | Europaeischer Standard fuer ICT-Barrierefreiheit | EU, oeffentliche Beschaffung |
| EU-Richtlinie 2016/2102 | Barrierefreiheit oeffentlicher Websites | EU, oeffentliche Stellen |
| BFSG (ab 2025) | Barrierefreiheitsstaerkungsgesetz | Deutschland, private Wirtschaft |

### WCAG-Prinzipien (POUR)

| Prinzip | Englisch | Bedeutung | Beispiel |
|---------|----------|-----------|----------|
| Wahrnehmbar | Perceivable | Inhalte muessen wahrgenommen werden koennen | Alternativtexte fuer Bilder |
| Bedienbar | Operable | Bedienung muss moeglich sein | Tastaturbedienbarkeit |
| Verstaendlich | Understandable | Inhalte muessen verstaendlich sein | Klare Sprache, konsistente Navigation |
| Robust | Robust | Inhalte muessen robust interpretierbar sein | Valides HTML, ARIA-Attribute |

### Datenschutz als ethische Pflicht

| Aspekt | Ethische Dimension |
|--------|-------------------|
| Datenminimierung | Nur erheben, was wirklich noetig ist – Respekt vor Privatsphaere |
| Zweckbindung | Daten nur fuer den angegebenen Zweck nutzen – Vertrauen |
| Transparenz | Betroffene informieren – Selbstbestimmung ermoeglichen |
| Loeschung | Daten nach Zweckerfuellung loeschen – Recht auf Vergessenwerden |
| Privacy by Design | Datenschutz von Anfang an einbauen – nicht nachtraeglich |

## TK 5: Net(t)ikette und unterschiedliche Perspektiven

**Definition:** Die Beruecksichtigung unterschiedlicher Perspektiven und Befindlichkeiten bei der digitalen Kommunikation erfordert Sensibilitaet, Empathie und die Einhaltung von Netiquette-Regeln.

### Netiquette-Grundregeln

| Regel | Beschreibung |
|-------|-------------|
| Menschlichkeit | Hinter jedem Bildschirm sitzt ein Mensch |
| Respekt | Keine Beleidigungen, Herabsetzungen oder Mobbing |
| Sachlichkeit | Kritik am Inhalt, nicht an der Person |
| Inklusion | Sprache verwenden, die niemanden ausschliesst |
| Gelassenheit | Nicht sofort emotional reagieren ("Sleep on it") |
| Kontext | Ton und Formalitaet dem Kanal anpassen |
| Privatsphaere | Keine privaten Informationen anderer veroeffentlichen |

### Unterschiedliche Perspektiven beruecksichtigen

| Perspektive | Beruecksichtigung in der IT |
|-------------|----------------------------|
| Alter | Interface-Design fuer alle Altersgruppen (Schriftgroesse, Kontrast) |
| Behinderung | Barrierefreiheit (Screenreader, Tastaturnavigation) |
| Kultur | Lokalisierung (Sprache, Symbole, Farbbedeutungen) |
| Geschlecht | Gender-neutrale Formulierungen und Designentscheidungen |
| Digitale Kompetenz | Einfache Bedienung, Hilfe-Funktionen, Onboarding |
| Neurodiversitaet | Reizarme Optionen, klare Strukturen, wenig Ablenkung |

# 4.5.01 – Wertschaetzende Zusammenarbeit: Grundlagen

## TK 1: Effektive, integre und respektvolle Zusammenarbeit in Teams

**Definition:** Wertschaetzende Zusammenarbeit beschreibt die Faehigkeit, in unterschiedlich zusammengesetzten Teams produktiv, ehrlich und respektvoll zu arbeiten, um gemeinsame Ziele zu erreichen.

### Grundbegriffe

| Begriff | Erklaerung |
|---------|------------|
| Effektivitaet | Die richtigen Dinge tun – Zielerreichung steht im Vordergrund |
| Integritaet | Ehrlichkeit, Zuverlaessigkeit und Uebereinstimmung von Worten und Handeln |
| Respekt | Achtung vor anderen Personen, ihren Meinungen und Beitraegen |
| Teamarbeit | Gemeinsames, koordiniertes Arbeiten an einem Ziel |
| Kollaboration | Zusammenarbeit auf Augenhoehe, gemeinsame Erarbeitung von Ergebnissen |
| Kooperation | Arbeitsteilige Zusammenarbeit mit definierten Schnittstellen |

### Teamrollen in IT-Projekten (Scrum)

| Rolle | Aufgabe | Beitrag zur Zusammenarbeit |
|-------|---------|---------------------------|
| Product Owner | Anforderungen priorisieren, Vision vermitteln | Klare Kommunikation, Entscheidungen treffen |
| Scrum Master | Hindernisse beseitigen, Prozess sichern | Moderation, Konfliktloesung, Teamschutz |
| Development Team | Umsetzung der User Stories | Selbstorganisation, gegenseitige Unterstuetzung |
| Stakeholder | Anforderungen einbringen, Feedback geben | Transparente Kommunikation, konstruktives Feedback |

### Kommunikationsformen im Team

| Form | Beispiel | Einsatz |
|------|----------|---------|
| Synchron | Daily Stand-up, Videokonferenz | Schnelle Abstimmung, Problemloesung |
| Asynchron | E-Mail, Chat, Wiki | Dokumentation, nicht-dringende Themen |
| Formal | Sprint Review, Protokoll | Offizielle Entscheidungen, Nachvollziehbarkeit |
| Informal | Kaffeekueche, Chat | Beziehungsaufbau, schneller Austausch |

## TK 2: Interdisziplinaritaet und Interkulturalitaet

**Definition:** Interdisziplinaritaet bezeichnet die Zusammenarbeit von Fachleuten verschiedener Disziplinen. Interkulturalitaet beschreibt das Zusammenwirken von Menschen unterschiedlicher kultureller Hintergruende.

### Dimensionen der Vielfalt in IT-Teams

| Dimension | Beispiele | Relevanz fuer IT-Projekte |
|-----------|-----------|--------------------------|
| Fachlich | Entwickler, Designer, Tester, Admins | Unterschiedliche Perspektiven auf Loesungen |
| Kulturell | Verschiedene Laender, Sprachen, Traditionen | Internationale Projekte, Offshore-Teams |
| Generational | Juengere und aeltere Mitarbeitende | Unterschiedliche Technologie-Affinitaet |
| Ausbildung | Studium, Ausbildung, Quereinsteiger | Verschiedene Herangehensweisen |

### Interkulturelle Kompetenz

| Kompetenzbereich | Beschreibung | Praxisbeispiel |
|-----------------|--------------|----------------|
| Wissen | Kenntnis kultureller Unterschiede | Zeitzonen, Feiertage, Kommunikationsstile |
| Haltung | Offenheit, Neugier, Toleranz | Andere Arbeitsweisen akzeptieren |
| Verhalten | Anpassung des eigenen Verhaltens | Direkte vs. indirekte Kommunikation |
| Reflexion | Eigene kulturelle Praegung erkennen | Eigene Annahmen hinterfragen |

### Herausforderungen bei internationalen IT-Projekten

| Herausforderung | Loesung |
|----------------|---------|
| Unterschiedliche Zeitzonen | Ueberlappende Kernarbeitszeiten definieren |
| Sprachbarrieren | Gemeinsame Arbeitssprache festlegen, Glossar erstellen |
| Kulturelle Missverstaendnisse | Interkulturelle Schulungen, explizite Kommunikation |
| Unterschiedliche Arbeitsstile | Gemeinsame Prozesse und Werkzeuge vereinbaren |

## TK 3: Unternehmenswerte und betriebliche Ethikregeln

**Definition:** Unternehmenswerte sind die grundlegenden Ueberzeugungen und Prinzipien, die das Handeln eines Unternehmens leiten. Betriebliche Ethikregeln sind verbindliche Verhaltensrichtlinien fuer alle Mitarbeitenden.

### Bestandteile eines Unternehmensleitbilds

| Element | Beschreibung | Beispiel |
|---------|--------------|---------|
| Vision | Langfristiges Zukunftsbild | "Wir gestalten die digitale Zukunft" |
| Mission | Grundauftrag des Unternehmens | "Sichere IT-Loesungen fuer den Mittelstand" |
| Werte | Handlungsleitende Prinzipien | Integritaet, Innovation, Kundenorientierung |
| Verhaltensregeln | Konkrete Handlungsanweisungen | Code of Conduct, Compliance-Richtlinien |

### Code of Conduct – Typische Inhalte

| Bereich | Regelung |
|---------|----------|
| Korruptionspraevention | Keine Bestechung, Geschenke nur bis Wertgrenze |
| Datenschutz | Vertraulicher Umgang mit Daten, DSGVO-Konformitaet |
| Gleichbehandlung | Keine Diskriminierung, Diversity foerdern |
| Interessenkonflikte | Offenlegung von Nebentaetigkeiten und Beteiligungen |
| Kommunikation | Respektvoller Umgang, keine Diffamierung |
| IT-Sicherheit | Verantwortungsvoller Umgang mit Systemen und Daten |

## TK 4: Gemeinsame Verantwortung und Wertschaetzung

**Definition:** Gemeinsame Verantwortung (Shared Ownership) bedeutet, dass alle Teammitglieder fuer das Gesamtergebnis verantwortlich sind. Wertschaetzung bezeichnet die Anerkennung der individuellen Beitraege jedes Teammitglieds.

### Agile Werte (Scrum)

| Wert | Bedeutung | Praxisbeispiel |
|------|-----------|----------------|
| Respekt | Achtung vor Faehigkeiten und Meinungen anderer | Konstruktives Code Review, keine Schuldzuweisungen |
| Mut | Schwierige Themen ansprechen, neue Wege gehen | Technische Schulden benennen, Fehler eingestehen |
| Offenheit | Transparenz ueber Arbeit und Herausforderungen | Impediments im Daily ansprechen, Burndown offen |
| Fokus | Konzentration auf die vereinbarten Sprint-Ziele | Sprint-Backlog respektieren, Scope Creep vermeiden |
| Commitment | Selbstverpflichtung zu Zielen und Qualitaet | Sprint-Ziel als Team-Versprechen |

### Feedbackkultur

| Feedbackform | Beschreibung | Einsatz |
|-------------|--------------|---------|
| Konstruktives Feedback | Konkret, sachlich, verbesserungsorientiert | Code Reviews, Pair Programming |
| Retrospektive | Team reflektiert Sprint-Verlauf | Sprint-Ende: Was lief gut? Was verbessern? |
| Peer Feedback | Feedback unter Gleichgestellten | 360-Grad-Feedback, Teamreflexion |
| Appreciative Inquiry | Staerkenorientierter Ansatz | Erfolge wuerdigen, Best Practices teilen |

### Feedbackregeln

| Regel | Erklaerung |
|-------|------------|
| Ich-Botschaften | "Ich habe beobachtet..." statt "Du machst immer..." |
| Konkret | Auf spezifische Situationen beziehen |
| Zeitnah | Moeglichst bald nach dem Ereignis |
| Konstruktiv | Verbesserungsvorschlaege machen |
| Wertschaetzend | Auch Positives benennen |
| Vertraulich | Kritik nicht oeffentlich aeussern |

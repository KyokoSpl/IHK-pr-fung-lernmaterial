# 4.5.02 – Informationstechnische Schutzziele bei der Kommunikation: Grundlagen

## TK 1: Social-Media-Gefahren fuer die IT-Sicherheit

**Definition:** Social-Media-Gefahren umfassen alle Risiken, die durch die Nutzung sozialer Netzwerke fuer die IT-Sicherheit eines Unternehmens oder einer Privatperson entstehen koennen.

### Gefahren durch Social Media

| Gefahr | Beschreibung | Beispiel |
|--------|-------------|----------|
| Informationsabfluss | Unbewusste Preisgabe interner Informationen | Foto vom Arbeitsplatz mit sichtbaren Bildschirmen |
| Phishing ueber Social Media | Gefaelschte Nachrichten/Links in sozialen Netzwerken | Fake-LinkedIn-Nachricht mit Malware-Link |
| Identitaetsdiebstahl | Uebernahme oder Faelschung von Profilen | Fake-Profil eines Vorgesetzten |
| Reputationsschaden | Negative Aeusserungen schaedigen Unternehmensimage | Mitarbeiter-Beschwerden ueber Arbeitgeber |
| Social Profiling | Angreifer sammeln Infos fuer gezielte Angriffe | OSINT-Recherche ueber Mitarbeiter |
| Malware-Verbreitung | Schaedliche Links/Dateien in Beitraegen | Virale Posts mit eingebetteter Malware |

### Social-Media-Policy – Typische Inhalte

| Element | Regelung |
|---------|----------|
| Private Nutzung | Erlaubt/eingeschraenkt/verboten waehrend Arbeitszeit |
| Dienstliche Nutzung | Wer darf offiziell im Namen des Unternehmens posten |
| Vertraulichkeit | Keine internen Informationen veroeffentlichen |
| Meinungsaeusserung | Privatmeinung kennzeichnen, Loyalitaetspflicht |
| Bilder/Videos | Keine Fotos von Arbeitsplaetzen, Servern, Bildschirmen |
| Kennzeichnung | "Meine persoenliche Meinung" bei privaten Aeusserungen |

## TK 2: Social Engineering

**Definition:** Social Engineering ist die Manipulation von Personen, um vertrauliche Informationen zu erlangen, Zugang zu Systemen zu erhalten oder Handlungen auszuloesen, die der Sicherheit schaden.

### Angriffsarten

| Methode | Beschreibung | Beispiel |
|---------|-------------|----------|
| Phishing | Gefaelschte E-Mails/Websites | "Ihr Konto wurde gesperrt – klicken Sie hier" |
| Spear-Phishing | Gezieltes Phishing auf bestimmte Person | E-Mail an CEO mit personalisierten Details |
| Whaling | Phishing auf Fuehrungskraefte | Gefaelschte Vorstandsanfrage |
| Pretexting | Erfundene Geschichte/Identitaet | "Ich bin der neue IT-Admin, brauche Ihr Passwort" |
| Baiting | Koeder mit Schadsoftware | USB-Stick auf Parkplatz mit "Gehaltsliste.xlsx" |
| Tailgating | Unbefugtes Mitgehen durch gesicherte Tueren | "Koennen Sie mir die Tuer aufhalten?" |
| Vishing | Voice-Phishing per Telefon | Anruf als "Microsoft-Support" |
| Quid pro Quo | Gegenleistung anbieten | "Kostenloses WLAN-Audit" gegen Zugangsdaten |
| Dumpster Diving | Durchsuchen von Abfaellen | Ausgedruckte Passwoerter im Papierkorb |

### Schutzmassnahmen

| Massnahme | Wirkung |
|-----------|---------|
| Security-Awareness-Schulungen | Mitarbeiter erkennen Angriffe |
| Phishing-Simulationen | Praxisnahe Uebungen erhoehen Wachsamkeit |
| Vier-Augen-Prinzip | Kritische Aktionen benoetigen Bestaetigung |
| Rueckruf-Verfahren | Identitaet telefonisch verifizieren |
| Clean-Desk-Policy | Keine sensiblen Unterlagen sichtbar |
| Besucherrichtlinien | Begleitung, Ausweispflicht, keine Alleingaenge |

## TK 3: Sicherheitsbewusstsein bei IT-Nutzung

**Definition:** Sicherheitsbewusstsein (Security Awareness) bezeichnet das Wissen und die Einstellung von Mitarbeitenden bezueglich IT-Sicherheitsrisiken und angemessenem Verhalten.

### Grundlegende Sicherheitsregeln

| Bereich | Regel |
|---------|-------|
| Passwoerter | Stark (Laenge > Komplexitaet), einzigartig, Passwort-Manager |
| Updates | Betriebssystem und Software aktuell halten |
| E-Mails | Absender pruefen, Links nicht blind anklicken |
| USB-Geraete | Unbekannte Datentraeger nicht anschliessen |
| Bildschirmsperre | Bei Verlassen des Arbeitsplatzes sperren (Win+L) |
| WLAN | Kein oeffentliches WLAN fuer dienstliche Zwecke ohne VPN |
| Software | Nur genehmigte Software installieren (Whitelist) |
| Sicherheitsvorfaelle | Sofort melden, nicht selbst "loesen" |

## TK 4: Juristische Konsequenzen von Aeusserungen ueber den Arbeitgeber

**Definition:** Arbeitnehmer unterliegen einer arbeitsvertraglichen Loyalitaetspflicht, die auch im digitalen Raum gilt. Aeusserungen ueber den Arbeitgeber in sozialen Netzwerken koennen arbeitsrechtliche Konsequenzen haben.

### Rechtlicher Rahmen

| Aspekt | Regelung |
|--------|----------|
| Loyalitaetspflicht | Nebenpflicht aus Arbeitsvertrag (§ 241 Abs. 2 BGB) |
| Meinungsfreiheit | Art. 5 GG – gilt auch fuer Arbeitnehmer |
| Grenzen | Schmaehurkritik, unwahre Tatsachenbehauptungen, Geschaeftsgeheimnisse |
| Konsequenzen | Abmahnung, Kuendigung (ordentlich/ausserordentlich) |
| Beweislast | Arbeitgeber muss Pflichtverletzung nachweisen |

### Was ist erlaubt, was nicht?

| Handlung | Erlaubt? | Begruendung |
|----------|----------|-------------|
| Sachliche Kritik am Arbeitgeber | Ja (eingeschraenkt) | Meinungsfreiheit, aber Loyalitaetspflicht beachten |
| Beleidigungen des Chefs | Nein | Schmaehurkritik, Kuendigungsgrund |
| "Mein Job nervt" (privat) | Grauzone | Abhaengig von Reichweite und Kontext |
| Internas teilen ("Wir haben wenig Auftraege") | Nein | Geschaeftsgeheimnis, Schadensersatz moeglich |
| Betriebsrat-Arbeit oeffentlich machen | Eingeschraenkt | Nur wenn keine Vertraulichkeit verletzt wird |
| Positive Bewertung auf Kununu | Ja | Aber muss wahrheitsgemaeß sein |

## TK 5: Zustaendigkeitsabgrenzung bei Kommunikation

**Definition:** Zustaendigkeitsabgrenzung regelt, wer in einem Unternehmen welche Informationen an wen kommunizieren darf (Need-to-Know-Prinzip).

### Kommunikationswege im Unternehmen

| Kommunikationsart | Zustaendig | Beispiel |
|-------------------|-----------|----------|
| Offizielle Pressemitteilung | Pressestelle / Marketing | Produktlaunch, Unternehmenskrise |
| Kundeninformation | Vertrieb / Kundensupport | Stoerungsmeldung, neues Feature |
| Interne IT-Sicherheit | CISO / IT-Sicherheitsbeaufragter | Sicherheitsvorfall, Patch-Info |
| Personalangelegenheiten | HR / Personalabteilung | Kuendigungen, Gehaltsthemen |
| Technische Dokumentation | Entwicklerteam / Techwriter | API-Dokumentation, Handbuecher |
| Eskalation | Definierter Eskalationsweg | Problem → Teamlead → Abteilungsleiter |

## TK 6: Nachrichten aus Empfaengersicht betrachten

**Definition:** Perspektivwechsel in der Kommunikation bedeutet, vor dem Senden einer Nachricht zu pruefen, wie diese beim Empfaenger ankommen koennte.

### Vier-Seiten-Modell (Schulz von Thun)

| Seite | Beschreibung | Beispiel: "Die Datei ist noch nicht fertig" |
|-------|-------------|---------------------------------------------|
| Sachinhalt | Wortueber informiert wird | Die Datei ist nicht abgeschlossen |
| Selbstoffenbarung | Was der Sender ueber sich verraet | Ich bin ungeduldig / besorgt |
| Beziehung | Wie Sender und Empfaenger zueinander stehen | Ich kontrolliere dich / Ich vertraue dir |
| Appell | Was der Sender bewirken will | Mach die Datei fertig / Brauchst du Hilfe? |

## TK 7: Reflexion von Erfahrungen in virtuellen Raeumen

**Definition:** Systematische Auswertung der Zusammenarbeit in digitalen Umgebungen, um die virtuelle Teamarbeit kontinuierlich zu verbessern.

### Reflexionsbereiche

| Bereich | Fragen |
|---------|--------|
| Werkzeuge | Sind unsere Tools geeignet? (Chat, Video, Wiki) |
| Kommunikation | Erreichen unsere Nachrichten alle? Gibt es Informationsluecken? |
| Erreichbarkeit | Sind Kernzeiten definiert? Wird asynchrone Arbeit respektiert? |
| Vertrauen | Fuehlen sich alle eingebunden? Gibt es "Schattenteams"? |
| Ergebnisse | Ist die Produktivitaet vergleichbar mit Praesenzarbeit? |

## TK 8: Sensibler Umgang mit Adressatenlisten

**Definition:** Der sorgsame Umgang mit E-Mail-Verteilern und Adressatenlisten zum Schutz personenbezogener Daten und zur Vermeidung ungewollter Informationsweitergabe.

### BCC vs. CC vs. TO

| Feld | Bedeutung | Wann verwenden? |
|------|-----------|-----------------|
| TO (An) | Hauptempfaenger, Handlung erwartet | Direkte Ansprechpartner |
| CC (Kopie) | Zur Kenntnis, keine Handlung erwartet | Vorgesetzte, Beteiligte zur Info |
| BCC (Blindkopie) | Empfaenger fuer andere unsichtbar | Newsletter, externe Verteiler, DSGVO |

### DSGVO-Relevanz

| Situation | Regelung |
|-----------|----------|
| Newsletter an Kunden | BCC verwenden oder professionelles Newsletter-Tool |
| Rundmail an externe Partner | BCC – E-Mail-Adressen sind personenbezogene Daten |
| Interne Teammail | CC ist in der Regel akzeptabel |
| Weiterleitung mit Verlauf | Pruefen, ob vertrauliche Inhalte enthalten sind |

## TK 9: Sicherer Umgang mit dienstlichen E-Mails / Net(t)ikette

**Definition:** Net(t)ikette bezeichnet die Verhaltensregeln fuer die hoefliche und respektvolle Kommunikation im Internet und in dienstlichen E-Mails.

### E-Mail-Sicherheit

| Massnahme | Beschreibung |
|-----------|-------------|
| S/MIME | Standardisierte E-Mail-Verschluesselung und -Signatur mit Zertifikaten |
| PGP / GPG | Asymmetrische Verschluesselung, Web-of-Trust-Modell |
| Digitale Signatur | Garantiert Authentizitaet und Integritaet der E-Mail |
| TLS/STARTTLS | Transportverschluesselung zwischen Mailservern |
| Spam-Filter | Filtert unerwuenschte und potentiell gefaehrliche E-Mails |
| Attachment-Scanning | Pruelung von Anhaengen auf Schadsoftware |

### Netiquette-Regeln fuer dienstliche E-Mails

| Regel | Erlaeuterung |
|-------|-------------|
| Aussagekraeftiger Betreff | Thema sofort erkennbar, z.B. "[Projekt X] Sprint Review Protokoll" |
| Anrede und Gruss | Professionell, dem Kontext angemessen |
| Klare Struktur | Kurze Absaetze, Aufzaehlungen bei mehreren Punkten |
| Keine Grossbuchstaben | VERSALIEN = Schreien |
| Antwortzeit beachten | Innerhalb eines Arbeitstages reagieren |
| Reply-All sparsam | Nur wenn alle die Antwort benoetigen |
| Anhanggroesse | Grosse Dateien per Cloud-Link teilen |
| Signatur | Kontaktdaten, Impressumspflicht bei geschaeftlichen Mails |
| Emoticons | Sparsam im geschaeftlichen Kontext |

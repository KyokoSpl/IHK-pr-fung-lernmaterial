# Ueberblick: 3.4.07 – Software-Entwicklungswerkzeuge aufgabenbezogen auswaehlen

## Einordnung

- **Pruefungsteil:** 3.4 – Konzipieren und Umsetzen von kundenspezifischen Softwareanwendungen
- **Thema-ID:** 3.4.07
- **Thema:** Software-Entwicklungswerkzeuge aufgabenbezogen auswaehlen und anwenden koennen

## Ziel

Du musst die verschiedenen Werkzeuge der Softwareentwicklung kennen, ihren Zweck im Entwicklungsprozess einordnen und begruenden koennen, wann welches Werkzeug eingesetzt wird.

## Themenkreise im Ueberblick

### 1. Compiler
Ein Compiler uebersetzt den gesamten Quellcode einer Programmiersprache in Maschinencode (oder Bytecode), bevor das Programm ausgefuehrt wird. Fehler werden zur Compile-Zeit erkannt.

### 2. Debugger
Ein Debugger ist ein Werkzeug zur systematischen Fehlersuche in Programmen. Er erlaubt das schrittweise Ausfuehren von Code, Setzen von Breakpoints und Inspizieren von Variablenwerten zur Laufzeit.

### 3. Editor
Ein Editor ist ein leichtgewichtiges Programm zum Schreiben und Bearbeiten von Quellcode. Moderne Editoren bieten Syntaxhervorhebung und Erweiterungen (z.B. VS Code, Sublime Text, Vim).

### 4. IDE
Eine IDE (Integrated Development Environment) buendelt Editor, Compiler/Interpreter, Debugger, Build-Tools und weitere Werkzeuge in einer einzigen Anwendung (z.B. IntelliJ IDEA, Visual Studio, Eclipse).

### 5. Interpreter
Ein Interpreter fuehrt den Quellcode zeilenweise zur Laufzeit aus, ohne ihn vorher vollstaendig in Maschinencode zu uebersetzen. Fehler werden erst zur Laufzeit erkannt.

### 6. Linker
Ein Linker verbindet kompilierte Objektdateien und Bibliotheken zu einem ausfuehrbaren Programm. Er loest externe Referenzen auf und erzeugt die finale Programmdatei.

### 7. Programmgenerator
Ein Programmgenerator erzeugt automatisch Quellcode aus Modellen, Vorlagen oder Spezifikationen (z.B. ORM-Code aus Datenbankschema, GUI-Code aus WYSIWYG-Editor).

### 8. Testsoftware
Testsoftware automatisiert das Testen von Programmen. Dazu gehoeren Unit-Test-Frameworks (JUnit, pytest), Integrationstests, Lasttests und GUI-Tests.

### 9. Versionsverwaltung
Versionsverwaltungssysteme (z.B. Git) protokollieren alle Aenderungen am Quellcode, ermoeglichen paralleles Arbeiten im Team und das Zuruecksetzen auf fruehere Versionen.

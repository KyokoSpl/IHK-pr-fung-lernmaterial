# Ueberblick: 3.4.17 – OOP-Methodenkonzepte anwenden

## Einordnung

- **Pruefungsteil:** 3.4 – Konzipieren und Umsetzen von kundenspezifischen Softwareanwendungen
- **Thema-ID:** 3.4.17
- **Thema:** OOP-Methodenkonzepte anwenden koennen

## Ziel

Du musst die Grundpfeiler der objektorientierten Programmierung (OOP) verstehen und anwenden koennen: Kapselung, Vererbung, Polymorphie, Interfaces und Fehlerbehandlung. Dazu gehoeren UML-Notation, Zugriffsmodifikatoren und das Schreiben von Pseudocode mit OOP-Konzepten.

## Themenkreise im Ueberblick

### 1. Allgemeine Fehlerbehandlung
Fehler (Exceptions) muessen im Programmcode abgefangen und sinnvoll behandelt werden. Dazu dient das try-catch-Konzept. Programme sollen nicht abstuerzen, sondern Fehler kontrolliert verarbeiten.

### 2. Interfaces
Ein Interface definiert eine Schnittstelle: Es legt fest, WELCHE Methoden eine Klasse implementieren muss, aber nicht WIE. Interfaces ermoeglichen lose Kopplung und Austauschbarkeit von Komponenten.

### 3. Kapselung
Kapselung (Encapsulation) bedeutet, dass interne Daten eines Objekts vor direktem Zugriff geschuetzt werden. Der Zugriff erfolgt ueber definierte Methoden (Getter/Setter). Zugriffsmodifikatoren (public, private, protected) steuern die Sichtbarkeit.

### 4. Polymorphie
Polymorphie (Vielgestaltigkeit) erlaubt es, dass Objekte verschiedener Klassen ueber eine gemeinsame Schnittstelle angesprochen werden koennen. Unterschieden werden Ueberladen (Overloading) und Ueberschreiben (Overriding).

### 5. Vererbung
Vererbung ermoeglicht es, dass eine Klasse (Kindklasse) Attribute und Methoden einer anderen Klasse (Elternklasse) uebernimmt. Das foerdert Wiederverwendbarkeit und Hierarchiebildung.
